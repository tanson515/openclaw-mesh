## 关键经验教训（今日新增）

### 1. 部署验证流程（硬性规则）
**必须在任何部署任务前读取此流程**

```
步骤1: 检查现有进程
  - 执行: ps aux | grep "服务进程名"
  - 结果: 记录现有PID
  
步骤2: 停止旧进程
  - 执行: pkill -f "服务进程名" 或 kill PID
  - 验证: ps aux 确认已停止
  
步骤3: 清理旧目录
  - 执行: rm -rf 旧目录
  - 验证: ls 确认已删除
  
步骤4: 解压/部署新代码
  - 执行: tar -xzf 或 git clone
  - 验证: ls 新目录确认文件存在
  
步骤5: 配置检查
  - 检查: config.yaml 或 .env
  - 修改: 必要的配置项
  
步骤6: 启动服务
  - 执行: nohup 命令 &
  - 记录: 新PID
  
步骤7: 验证运行状态
  - 检查1: ps aux | grep 进程名
  - 检查2: netstat -tlnp | grep 端口
  - 检查3: tail -20 日志文件
  
步骤8: 汇报结果
  - 状态: 成功/失败
  - PID: 新进程号
  - 端口: 监听状态
  - 日志: 关键信息
```

**硬性规则**:
- 每个步骤 timeout <= 30 秒
- 超时立即停止，禁止自动重试
- 用户指令"停止"立即停止所有操作
- 完成一步，验证一步，汇报一步

### 2. 工具执行规则
- **超时时间**: 所有 exec 命令 timeout=30 秒
- **超时后**: 立即停止，汇报"超时，等待用户决策"
- **重试**: 禁止自动重试，必须用户明确指示

### 3. 消息发送规则
- 禁止刷屏：同一内容绝不发送两次
- 分段汇报：长任务分步骤，每步完成后汇报
- 飞书通道：重要消息必须使用 message 工具
